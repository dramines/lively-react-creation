Introduction

L'API JenCity fournit des interfaces pour accéder aux données touristiques, gérer les utilisateurs, les lieux d'intérêt, les événements, les réservations, les avis et la communication entre utilisateurs.

Cette documentation détaille toutes les routes disponibles, les paramètres requis, et les formats de réponse.

Base URL

Toutes les requêtes doivent être préfixées avec l'URL de base :

http://localhost:3000

Authentification

La plupart des routes nécessitent une authentification. Après vous être connecté avec la route /api/users/login, vous recevrez un token JWT qu'il faudra inclure dans vos requêtes suivantes :

Authorization: Bearer votre_token_jwt

Utilisateurs

Entité User

L'entité User représente les utilisateurs de l'application. Elle contient les informations de profil et d'authentification.

Attributs

Attribut Type Description
user_id number Identifiant unique
email string Adresse email unique
password string Mot de passe (hashé)
nom string Nom de famille
prenom string Prénom
role string Rôle (user, provider, admin)
profile_image string URL de l'image de profil
created_at date Date de création
updated_at date Date de dernière mise à jour
POST
/api/users/register
Créer un nouveau utilisateur

Description

Crée un nouveau compte utilisateur et retourne les informations de base de l'utilisateur avec un token JWT.

Paramètres

Cette route n'utilise pas de paramètres d'URL.

Corps de la Requête

Paramètre Type Description
nom Requis string Nom de famille
prenom Requis string Prénom
email Requis string Adresse email (doit être unique)
password Requis string Mot de passe (min. 6 caractères) 
{
  "nom": "Dupont",
  "prenom": "Jean",
  "email": "jean.dupont@example.com",
  "password": "motdepasse123"
}

Réponses

201 Created 
{
  "status": 201,
  "message": "Utilisateur créé avec succès",
  "data": {
    "user_id": 1,
    "nom": "Dupont",
    "prenom": "Jean",
    "email": "jean.dupont@example.com",
    "role": "user",
    "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
  }
}
400 Bad Request 
{
  "status": 400,
  "message": "Validation error",
  "errors": [
    {
      "field": "email",
      "message": "Email déjà utilisé"
    }
  ]
}

Réservations

Entité Reservation

L'entité Reservation permet de gérer les réservations pour les événements et les visites de lieux.

Attributs

Attribut Type Description
id number Identifiant unique
userId number ID de l'utilisateur
eventId number ID de l'événement (optionnel)
placeId number ID du lieu (optionnel)
numberOfTickets number Nombre de billets (pour événements)
numberOfPersons number Nombre de personnes (pour visites)
totalPrice number Prix total
status string Statut (pending, confirmed, cancelled)
paymentMethod string Méthode de paiement
paymentId string ID de la transaction de paiement
visitDate date Date de visite (pour les lieux)
createdAt date Date de création
updatedAt date Date de dernière mise à jour

Description

Nom Valeur Description

Messagerie

Entité Messagerie

L'entité Messagerie permet aux utilisateurs d'échanger des messages dans l'application.

Attributs Message

Attribut Type Description
id number Identifiant unique
id_expediteur number ID de l'expéditeur
id_destinataire number ID du destinataire
texte string Contenu du message
is_read boolean Statut de lecture
date_envoi date Date d'envoi

Attributs Session

Attribut Type Description
id number Identifiant unique
userId1 number ID du premier utilisateur
userId2 number ID du second utilisateur
isActive boolean Statut de la session
createdAt date Date de création
updatedAt date Date de dernière mise à jour
POST
/api/messagerie
Envoyer un message

Description

Envoie un message à un autre utilisateur. Si aucune session n'existe entre les utilisateurs, une nouvelle session est automatiquement créée.

En-têtes

Nom Valeur Description

Paramètre Type Description

}
